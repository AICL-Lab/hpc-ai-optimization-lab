# API Reference

Welcome to the HPC-AI-Optimization-Lab API reference.

## Overview

This project provides a comprehensive set of optimized CUDA kernels for AI workloads, organized into logical modules.

## Module APIs

### Core Modules

| Module | Description | Status |
|--------|-------------|--------|
| [Elementwise](#elementwise) | Element-wise operations (ReLU, Sigmoid, Transpose, VectorAdd) | ✅ Stable |
| [Reduction](#reduction) | Reduction operations (Softmax, LayerNorm, RMSNorm) | ✅ Stable |
| [GEMM](#gemm) | Matrix multiplication with 7 optimization levels | ✅ Stable |
| [Convolution](#convolution) | Convolution kernels (implicit GEMM, Winograd) | ✅ Stable |
| [Attention](#attention) | Attention mechanisms (FlashAttention, RoPE, TopK) | ✅ Stable |
| [Quantization](#quantization) | Quantization kernels (INT8, FP8) | ✅ Stable |
| [CUDA 13](#cuda13) | Hopper architecture features (TMA, clusters, FP8) | 🧪 Experimental |

### Common Utilities

| Utility | Description |
|---------|-------------|
| `CudaTimer` | High-precision GPU timer |
| `Tensor<T>` | RAII GPU memory management |
| `CUDA_CHECK` | Error handling macro |
| `launch_config` | Automatic kernel launch configuration |

## Quick Reference

### Elementwise Operations

```cpp
#include "elementwise/relu.cuh"

hpc::module::relu(input, output, n, stream);
```

### Reduction Operations

```cpp
#include "reduction/softmax.cuh"

hpc::module::softmax(input, output, batch_size, seq_len, hidden_dim, stream);
```

### Matrix Multiplication

```cpp
#include "gemm/gemm.cuh"

hpc::module::gemm(A, B, C, M, N, K, stream);
```

## API Design Principles

1. **Unified Interface**: All kernels follow a consistent API pattern
2. **Template-based**: Support for multiple data types via templates
3. **Stream-aware**: All operations accept an optional CUDA stream
4. **Optimization Levels**: Choose between Basic, Intermediate, and Advanced

## Type Constraints

All kernels support the following data types:

| Type | Description | Support Level |
|------|-------------|---------------|
| `float` (FP32) | Single precision | ✅ Full |
| `__half` (FP16) | Half precision | ✅ Full |
| `__nv_bfloat16` | Brain float | 🧪 Partial |
| `__nv_fp8_e4m3` | 8-bit float (Hopper) | 🧪 Experimental |

## Error Handling

All API functions use `CUDA_CHECK` for error handling:

```cpp
#include "common/cuda_check.cuh"

// CUDA_CHECK automatically handles errors
hpc::module::relu(input, output, n);
```
