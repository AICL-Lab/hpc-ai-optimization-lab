Five Core Optimization Topics

From memory access fundamentals to cutting-edge CUDA 13 features.

Beginner · Week 1

Memory Optimization

Build intuition for global memory access patterns, shared memory reuse, vectorized loads, and cache behavior. For many CUDA kernels, memory access patterns become the bottleneck before arithmetic does.

  • Coalesced Access: Why it's the first priority for most kernels
  • Vectorization: float4 loads/stores and their throughput benefits
  • Shared Memory: When to use it for data reuse, avoiding bank conflicts
  • Grid Stride Loops: Handling arbitrary input sizes safely
Read Full Guide Related Source

Prerequisites

Basic understanding of thread blocks, warps, global memory, and shared memory concepts.

Next Steps

After this guide, read Reduction Optimization, then revisit GEMM tiling strategies.

Beginner to Intermediate · Week 1-2

Reduction Optimization

Reductions are excellent exercises for understanding CUDA parallel cooperation. They expose key issues: thread work distribution, synchronization costs, branch divergence, and shared memory layout.

  • Block vs Warp Reduction: Understanding the boundary between them
  • Warp Shuffle: Reducing synchronization overhead with __shfl_down_sync
  • Online Algorithms: Single-pass softmax without storing intermediate results
  • Occupancy: Building intuition for occupancy vs throughput tradeoffs
Read Full Guide Related Source

Why Learn This First

Softmax, layer norm, and similar operators are all reduction problems. Understanding them is essential for attention mechanisms.

Python API

hpc_ai_opt.reduction exposes softmax, layer_norm, and rms_norm.

Core Path · Week 2-4

GEMM Optimization

GEMM is not just matrix multiplication—it's the foundation for understanding convolution, attention, Tensor Core usage, register tiling, and tile design. Best approached after memory and reduction basics.

  • 7-Step Journey: Naive → Shared Memory → Double Buffer → Register Tiling → Tensor Core
  • CUTLASS Patterns: Understanding tile design philosophy
  • cuBLAS Comparison: Benchmarking against vendor libraries
  • Multi-Precision: SGEMM, HGEMM, Int8-GEMM support
Read Full Guide Related Source

Difficulty Note

Complete Steps 1-4 (memory + tiling) before attempting Steps 5-7 (Tensor Core).

Extension

The convolution module's implicit GEMM directly reuses tiling strategies from this guide.

Advanced · Week 3-4

FlashAttention

Focus on IO-aware design and online softmax. The repository implementation supports float precision with head_dim=64.

  • IO Analysis: Why standard attention writes O(N²) to HBM
  • Tiling Strategy: Keeping intermediate matrices in SRAM
  • Online Softmax: Derivation and implementation details
  • Warp Assignment: FlashAttention-2 style thread mapping
Read Full Guide Related Source

Prerequisites

Requires understanding of GEMM tiling and online reduction algorithms.

Current Limitations

Repository supports float precision and head_dim=64. Advanced variants not yet implemented.

Experimental · Advanced

CUDA 13 & Hopper Features

Explore TMA, Thread Block Clusters, and FP8 GEMM. Current implementations use fallback paths—these are educational examples, not full Hopper/Blackwell support.

  • TMA: Understanding the hardware async copy programming model
  • Clusters: Cross-SM thread block cooperation
  • FP8 GEMM: Low-precision matrix multiplication experiments
  • Fallback Behavior: What happens when features aren't available
Read Full Guide Related Source

Experimental Note

tma.cu, cluster.cu, fp8_gemm.cu use fallback implementations. No Hopper hardware required.
