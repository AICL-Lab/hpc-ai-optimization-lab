# Project Roadmap | 项目收尾方向

This repository is in a **simplify-and-harden** phase. The goal is to keep the current release surface credible, compact, and easy to maintain rather than to keep expanding repository process or feature breadth.

---

## Current focus | 当前重点

- simplify the repository surface and remove stale process scaffolding
- keep README, GitHub Pages, and API/docs copy aligned with shipped behavior
- retain only low-noise automation that protects real build or docs regressions
- remove duplicated history, dead branches of documentation, and trust-reducing drift

---

## Not a close-out priority | 当前不作为收尾优先项

- large new kernel families or broad API expansion
- speculative roadmap promises without measured implementation progress
- heavy workflow frameworks, plugin bundles, or repository-wide AI control layers
- new public claims that exceed the currently shipped C++/CUDA and Python surface

---

## Stabilization completion criteria | 稳定化完成标准

The current cleanup program is considered complete when:

- active repository documents describe the same shipped system boundaries
- GitHub-facing surfaces stay concise and internally consistent
- retained CI and Pages automation stay honest, focused, and easy to maintain
- remaining follow-up work fits into a small practical backlog instead of sprawling process machinery

---

## Remaining backlog | 剩余待办

- do one final repository-wide pass for stale claims, duplicate docs, and dead links
- keep tightening the Pages landing flow so it routes newcomers to guides, examples, and API references quickly
- validate retained local or self-hosted GPU verification paths when code changes require them
